一、这插件是干嘛的
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

打本的时候自动跟着一个人走。比如你跟坦克保持距离，
坦克跑你就跑，坦克停你也停。中间遇到怪会自动判断
要不要停下来让循环插件接手打怪。

全靠 vnavmesh 来走路，所以你装了 vnavmesh 才能用。


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
二、安装
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. 卫月设置 → 插件 → 本地插件 → 添加 DLL
2. 选 AutoFollow.dll
3. 启用

ZIP 里自带了 ImGui.NET.dll，放一起就行。


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
三、怎么用
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【开始跟随】

  游戏里点一下你要跟的人（选中他）
  聊天框打 /ftar
  他就变成你的跟随目标了，插件开始跟。

  /ft 小明    ← 直接打名字也行

【切换开关】

  /ff         切一下，跟 / 不跟 之间切换
  /fes        紧急停止，跟和循环全停了

【控制循环插件】

  /flp        暂停循环
  /flr        恢复循环
  /flt        切一下

  前提是你先在设置里填好命令：
    暂停命令: /rotation off
    恢复命令: /rotation Auto

【其他】

  /fst        聊天框里看状态
  /fdbg       打开主窗口


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
四、插件的运行逻辑（仔细看这个）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

整个插件干的事情很简单：
每隔 N 秒扫一次目标坐标，发给 vnavmesh 让它走过去。

N 默认为 2 秒，可以在设置里改，叫"坐标扫描间隔"。

┌─────────────────────────────────────────────┐
│  每2秒一次：                                  │
│    1. 看目标在哪里                             │
│    2. 跟上次比有没有动                         │
│    3. 动了超过0.5码 → 发坐标给 vnavmesh       │
│    4. vnavmesh 自己规划路径走过去              │
│    5. 没动 → 跳过，不打断 vnavmesh             │
└─────────────────────────────────────────────┘

【什么时候停、什么时候走】

  正常跟着 → 距离目标 ≤ 10y → 停！
    （暂停跟随、恢复循环插件，让输出循环接手）

  停下后 → 距离目标 > 30y → 走！
    （停止循环、恢复跟随，继续跟上去）

  停下后 → 脱战超过 1 秒 → 走！
    （不管距离多少，强制恢复跟随）

  每次恢复走的时候：
    → 立刻开疾跑（如果疾跑能用的话）
    → 立刻扫一次坐标（不等那个 2 秒的间隔）
    → 清掉上次的坐标缓存，强制发一次移动

  刚设好目标时 2 秒内不会因为距离近就停，
  免得你刚跟上就立刻停。

【Boss 房检测】

  目前没有做 Boss 房检测。
  全靠距离（10y/30y）+ 脱战 1 秒来判断。

五、主窗口
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

输入 /fdbg 打开，分四个翻页。

【状态栏】在窗口最顶部

  [跟随中/暂停中]  目标: 小明  距离: 12.3y  [战斗中/非战斗]

• 跟随中 / 暂停中 —— 插件当前的运行状态
• 战斗中 / 非战斗 —— 角色自己是不是在战斗状态
  （这两个是分开显示的）

【设置页】

  ┌─ 距离阈值 ────────────────────────────┐
  │  进入战斗区(y)     [ 10  ]             │
  │  离开战斗区(y)     [ 30  ]             │
  │  疾跑触发(y)       [ 20  ]             │
  │  坐标扫描间隔(秒)  [ 2   ]             │
  │                                         │
  │  [保存距离] [恢复默认]                   │
  └─────────────────────────────────────────┘

  • 进入战斗区：跟目标小于这个距离就暂停
  • 离开战斗区：跟目标大于这个距离就恢复
  • 疾跑触发：超过这个距离自动开疾跑
  • 扫描间隔：每隔几秒看一下目标位置

  ┌─ 疾跑设置 ────────────────────────────┐
  │  ✓ 启用疾跑                             │
  │  □ 仅战斗疾跑                           │
  │  □ 自动续疾跑                           │
  └─────────────────────────────────────────┘

  ┌─ 紧急停止热键 ─────────────────────────┐
  │  默认 F8，长按 300ms 触发               │
  └─────────────────────────────────────────┘

  ┌─ 循环插件命令 ─────────────────────────┐
  │  暂停命令: /rotation off              │
  │  恢复命令: /rotation Auto              │
  └─────────────────────────────────────────┘

  ┌─ 自定义命令 ───────────────────────────┐
  │  可以自己加命令、改命令                   │
  │  默认折叠，点开看                        │
  └─────────────────────────────────────────┘

【坐标页】

  实时看自己和目标的坐标。
  可以手动填 X Y Z 让 vnavmesh 走过去。

【插件状态页】

  点"重新检测"看 vnavmesh、BossMod 这些
  插件有没有装、有没有启用。

【指令日志页】

  插件每一步干了什么都在这里记着。
  按颜色分：
    绿色 = IPC 通信
    蓝色 = 移动相关
    黄色 = 状态切换
    白色 = 命令/其他

  支持按关键词过滤。

六、自定义命令
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

设置页最底下"自定义命令"点开，你可以：

  • 添加：打命令文本、选动作、写备注
  • 启用/禁用：临时关掉不需要的命令
  • 删除：去掉不要的

默认带这些：

  /flp      暂停循环插件
  /flr      恢复循环插件
  /flt      切换循环插件
  /ff       切换跟随开关
  /fst      输出状态报告
  /ft       设置跟随目标 <玩家名>
  /fes      紧急停止
  /fdbg     打开/关闭主窗口
  /ftar     跟随当前选中的目标

这些都可以自己改。

七、跟循环插件配合
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

设置里的"循环插件命令"填好命令：

  暂停命令: /rotation off
  恢复命令: /rotation Auto

具体填什么取决于你用什么循环插件。
AutoDuty 就填上面的。
如果是开关型的，只填暂停命令就行，恢复时也会
发暂停命令那一条。

工作流程：

  跟人走 → 距离 ≤ 10y → 暂停跟随
                       → 发恢复命令给循环插件
                       → 循环插件+BossMod 接手

  打完 → 脱战 1 秒    → 强制开疾跑
      → 距离 > 30y    → 立刻扫目标坐标
                       → 发暂停命令给循环插件
                       → 继续跟上去

八、可能的问题
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【加载不出来】

  • 检查 DalamudApiLevel 是不是 15
  • 检查 ImGui.NET.dll 有没有和 DLL 放一起
  • 去 /xllog 看看报错

【跟了不走】

  • 检查 vnavmesh 有没有装、有没有开
  • 看 vnavmesh 界面是不是绿色（说明地图加载好了）
  • 看指令日志有没有移动相关的记录
  • 去坐标页手动填坐标测试 vnavmesh 能不能走

【卡顿】

  • 是不是开了调试输出？关了试试
  • 插件逻辑是后台跑的，不影响主线程

【紧急停止没反应】

  • 检查热键有没有绑上
  • 是不是按得不够久（默认 300ms）
  • 别的插件是不是占了同一个键
