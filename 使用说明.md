## 安装方法

1. 解压 `AutoFollow_v1.3.3.zip`
2. 进入游戏，打开`卫月设置` → `插件` → `本地插件`
3. 点击`添加 DLL`，选择解压出来的 `AutoFollow.dll`
4. 在插件列表中找到 **强效跟随**，启用即可


### 依赖插件

| 插件 | 必要性 | 说明 |
|------|--------|------|
| **vnavmesh** | **必须** | 寻路导航，负责实际移动。从第三方仓库 `https://puni.sh/api/repository/veyn` 安装 |
| BossMod / BossModReborn | 可选 | 战斗辅助，进入战斗范围时 AutoFollow 自动暂停跟随让 BossMod 接管 |
| RotationSolverReborn | 可选 | 自动输出插件 |

## 快速开始

### 1. 跟随当前目标

```
操作流程：
  1. 在游戏中选中一个玩家（用鼠标点击他）
  2. 聊天框输入 /ftar
  3. 插件锁定该玩家为目标，开始自动跟随
```

> `/ftar` 锁定的是**输入命令那一刻选中的目标**，之后你切换目标打怪、做其他操作，都不会影响跟随对象。

### 2. 通过名字跟随

```
/ft 玩家名
```

例如 `/ft 小明` — 自动找到叫"小明"的玩家并开始跟随。

### 3. 切换跟随

```
/ff    切换跟随（开/关）
/fes   紧急停止（停止跟随 + 暂停循环插件）
```

## 跟随机制

### 核心逻辑

```
每1秒执行一次：
  1. 获取跟随目标的当前坐标
  2. 对比上次发送的坐标
  3. 如果移动超过0.5码 → 发送新目标坐标给 vnavmesh
  4. vnavmesh 自动规划路径并走过去
  5. 如果目标没动 → 跳过，不打断 vnavmesh
```

所有寻路计算在**后台线程**执行，不影响游戏帧率。

### 距离分区

| 距离 | 行为 |
|------|------|
| < 战斗区距离（默认 5y） | 暂停跟随，恢复循环插件，交给 BossMod 处理战斗 |
| 战斗区 ~ 最大跟随距离 | 正常跟随，每 1 秒发送坐标到 vnavmesh |
| > 最大跟随距离（默认 20y） | 继续跟随，进入追赶模式 |
| > 传送阈值（默认 40y） | 提示距离过远，暂停跟随 |

### 自动疾跑

超过设置的距离阈值（默认 10y）时自动使用疾跑技能追赶目标。

## 主窗口

插件加载后默认打开主窗口，包含四个翻页：

### 设置（默认页）

| 分组 | 说明 |
|------|------|
| 距离阈值 | 战斗区距离 / 停止距离 / 疾跑触发 / 最大跟随 / 传送阈值 |
| 疾跑设置 | 启用疾跑 / 仅战斗疾跑 / 自动续疾跑 |
| 行为开关 | 战斗中暂停 / 目标丢失暂停 / 死亡暂停 |
| 紧急停止热键 | 绑定按键 + 长按触发时长 |
| 循环插件命令 | 暂停/恢复循环插件的陌语命令内容 |
| 调试模式 | 调试输出开关 + 输出间隔 |
| 自定义命令 | 查看/添加/删除自定义陌语命令 |

### 坐标

实时显示：
- **玩家坐标** — 当前角色位置
- **跟随目标坐标** — 当前跟随目标的位置
- **手动移动** — 输入 X/Y/Z 坐标，点击"移动到坐标"让 vnavmesh 寻路过去
  - 也可以点"复制玩家坐标"快速填入当前位置

### 插件状态

点击"重新检测"扫描以下插件是否已安装并启用：

| 分类 | 检测项 |
|------|--------|
| 寻路导航 | vnavmesh |
| 战斗辅助 | BossMod, BossModReborn |
| 自动输出 | RotationSolverReborn, RotationSolver |

显示 **已连接 / 未连接** 状态。

### 指令日志

记录插件所有操作，按颜色分类：

| 颜色 | 分类 | 内容示例 |
|------|------|---------|
| 绿色 | IPC | vnavmesh 连接、寻路调用 |
| 蓝色 | move | 发送移动指令、目标移动 |
| 黄色 | state | 进入/离开战斗区、目标丢失 |
| 白色 | cmd | 设置目标、暂停/恢复、紧急停止 |

工具栏支持过滤关键词、清空日志。

## 命令列表

### 默认命令

| 命令 | 动作 | 说明 |
|------|------|------|
| `/ff` | ToggleFollow | 切换跟随开/关 |
| `/follow` | FollowCurrentTarget | 跟随当前选中的目标 |
| `/ftar` | FollowCurrentTarget | 同上 |
| `/ft <名>` | SetFollowTarget | 设置跟随目标（按名字） |
| `/fes` | EmergencyStop | 紧急停止 |
| `/flp` | PauseLoop | 暂停循环插件 |
| `/flr` | ResumeLoop | 恢复循环插件 |
| `/flt` | ToggleLoop | 切换循环插件 |
| `/fst` | StatusReport | 输出状态报告到聊天 |
| `/fdbg` | ToggleDebugWindow | 打开/关闭主窗口 |
| `/fcfg` | ToggleDebugWindow | 打开/关闭主窗口 |

### 自定义命令

在设置页的"自定义命令"面板中，可以：

1. **添加** — 输入命令文本、选择动作、填写说明
2. **启用/禁用** — 临时关闭不需要的命令
3. **删除** — 移除不需要的命令
4. **保存** — 修改后点击保存生效

### 全部动作列表

| 动作 | 说明 |
|------|------|
| PauseLoop | 发送陌语命令暂停循环插件 |
| ResumeLoop | 发送陌语命令恢复循环插件 |

## 设置说明

### 距离阈值

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 战斗区距离 | 5y | 进入此范围暂停跟随 |
| 停止距离 | 3y | 距离目标小于此值时停止移动 |
| 疾跑触发 | 10y | 超过此距离自动使用疾跑 |
| 最大跟随 | 20y | 超出此距离进入追赶模式 |
| 传送阈值 | 40y | 超出此距离暂停跟随并提示 |

### 疾跑设置

| 选项 | 说明 |
|------|------|
| 启用疾跑 | 总开关 |
| 仅战斗疾跑 | 仅在战斗状态中自动疾跑 |
| 自动续疾跑 | 疾跑结束后立刻再开 |

### 紧急停止热键

- 默认：**F8**，长按 **300ms**
- 触发后停止跟随 + 暂停循环插件
- 进入紧急停止状态，不会自动恢复，需要手动 `/ff` 恢复

## 与循环插件协同

### 配置方法

在"设置"页的"循环插件命令"中填写：

```
暂停命令: /rotation off
恢复命令: /rotation Auto
```

> 具体命令取决于你使用的循环插件。对于 RotationSolver，一般是 `/rotation off` 和 `/rotation Auto`。如果是开关型插件，只填暂停命令即可（恢复时也会发送暂停命令）。

### 工作流程

```
跟随中目标走远 → AutoFollow 暂停循环插件
                → 继续跟随（每1秒发坐标给 vnavmesh）
                → vnavmesh 绕开障碍物走过去

接近目标进入战斗区 → AutoFollow 暂停跟随
                   → 恢复循环插件（BossMod 接管战斗）
                   → 循环插件处理输出和机制

战斗结束目标继续前进 → AutoFollow 恢复跟随
                     → 暂停循环插件
                     → 继续跟随目标
```

## 故障排查

### 插件加载后不显示

1. 检查 `AutoFollow.json` 中 `DalamudApiLevel` 是否为 `15`（国服）
2. 检查 `ImGui.NET.dll` 是否和 `AutoFollow.dll` 在同一目录
3. 查看游戏日志（`/xllog`）是否有报错信息

### 跟随不动

1. 检查 vnavmesh 是否已安装并启用（主窗口 → 插件状态页查看）
2. 检查 vnavmesh 的 NavMesh 是否已加载（vnavmesh 窗口显示绿色）
3. 查看指令日志页是否有"move"记录
4. 尝试在坐标页手动输入坐标点"移动到坐标"测试 vnavmesh 是否正常工作

### 卡顿／掉帧

1. 检查是否开启了调试模式（调试输出默认 2 秒一次，不影响帧率）
2. 检查是否有其他插件冲突
3. 跟随逻辑已优化为后台线程执行，主线程不受影响

### 紧急停止不生效

1. 检查设置页的热键是否被正确绑定
2. 长按时间是否太短（默认 300ms）
3. 检查是否有其他插件占用了相同热键
