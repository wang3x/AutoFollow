# 强效跟随 AutoFollow

> 卫月 (Dalamud) 自动跟随插件 — 支持寻路避障、自动疾跑、循环插件协同

---

## 📦 安装

### 方式一：本地插件导入（推荐）

1. 解压 `AutoFollow_v1.1.0.zip`
2. 进入游戏，打开 `卫月设置` → `插件` → `本地插件`
3. 点击 `添加 DLL`，选择解压后的 `AutoFollow.dll`
4. 在插件列表中找到 **强效跟随**，启用即可

### 方式二：devPlugins 手动部署

1. 将 `AutoFollow.dll` 和 `AutoFollow.json` 放入：
   ```
   %APPDATA%\XIVLauncher\devPlugins\AutoFollow\
   ```
2. 重启游戏或在聊天框输入 `/xlload`
3. 打开插件管理器 (`/xlplugins`) → `开发工具` → `已安装的开发插件` → 启用

## 🎮 快速上手

### 1. 设置跟随目标

```
/ft <玩家名>
```

例如 `/ft 小明` — 插件会自动找到队伍中叫"小明"的玩家并开始跟随。

### 2. 开始/停止跟随

| 命令 | 作用 |
|------|------|
| `/ff` | 切换跟随（开/关） |
| `/fes` | 紧急停止（停止跟随 + 暂停循环插件） |
| `/ft <玩家名>` | 设置跟随目标 |

### 3. 控制循环插件

| 命令 | 作用 |
|------|------|
| `/flp` | 暂停自动循环插件 |
| `/flr` | 恢复自动循环插件 |
| `/flt` | 切换循环插件（暂停/恢复） |

### 4. 其他

| 命令 | 作用 |
|------|------|
| `/fst` | 查看当前状态报告 |
| `/ffm` | 切换跟随模式（原生/Vnavmesh/自动） |
| `/fdm` | 切换调试模式开/关 |

> 以上所有命令均可打开设置窗口，在 **自定义陌语命令** 面板中修改或增删。

## 🧠 核心机制

### 距离分区状态机

插件根据与目标的距离，自动切换行为模式：

```
距离 < 战斗区 (5y)     → 暂停跟随 + 恢复循环插件 → BossMod 接管
距离 < 跟随区 (10y)    → 正常跟随          
距离 > 疾跑阈值 (10y)  → 自动开疾跑追赶
距离 > 最大跟随 (20y)   → 暂停循环插件 + 加速追赶
距离 > 传送阈值 (40y)   → 提示用户手动传送
```

> 以上距离值均可在设置面板中调整。

### 跟随模式

**① 系统原生跟随**
- 直接朝目标直线移动
- 优点：无外部依赖，响应最快
- 缺点：不避障，遇墙会卡住

**② Vnavmesh 寻路跟随**
- 通过 vnavmesh 插件做路径规划，自动绕开障碍物
- 优点：自动避障，适合复杂地形
- 依赖：需要安装 [vnavmesh](https://puni.sh/api/repository/veyn) 插件

**③ 自动模式（默认）**
- 检测到 vnavmesh 时自动使用寻路模式，否则用原生模式

### 循环插件协同

插件通过 IPC 与 AutoDuty / BossMod / RotationSolver 等循环插件协作：

```
跟随中目标走远 (>20y)   → AutoFollow 暂停循环插件，追赶目标
接近目标进入战斗区 (<5y) → AutoFollow 暂停跟随，恢复循环插件
                        BossMod 接管战斗移动和动作
战斗结束目标继续前进      → AutoFollow 恢复跟随，暂停循环插件
```

#### 自定义 IPC 路径

如果你的循环插件不是 AutoDuty，而是自研或其他插件，可以在设置中手动填入 IPC 路径：

| 字段 | 示例 |
|------|------|
| 状态 IPC | `MyPlugin.State` |
| 暂停 IPC | `MyPlugin.Pause` |
| 恢复 IPC | `MyPlugin.Resume` |

填入后优先使用自定义路径，留空则自动扫描 AutoDuty/LoopPlugin 等常见前缀。

## ⚡ 紧急停止

**默认热键：长按 `F8` 300ms**

触发后：
- ❌ 立刻停止跟随
- ❌ 暂停循环插件
- 进入 **紧急停止** 状态（不会自动恢复）

恢复方式：
- 命令 `/ff` 切换回跟随
- 命令 `/flr` 手动恢复循环插件

可在设置面板中修改热键和长按时长（100ms–2000ms）。

## 🐛 调试模式

设置中打开 **调试模式** 后，插件会每隔 N 秒向聊天频道输出当前状态：

```
[跟随调试] 状态:Following | 距离:12.3y | 区域:追赶区 | 模式:Vnavmesh寻路 | 疾跑:True | 暂停条件: | 循环:Running
```

可用于排查插件工作是否正常。命令 `/fdm` 可快速开关。

## 🛠 设置界面一览

在游戏中输入 `/xlplugins` → 选中 **强效跟随** → `设置`：

| 面板 | 说明 |
|------|------|
| 距离阈值 | 战斗区 / 跟随区 / 疾跑 / 最大跟随 / 传送阈值 |

| 紧急停止热键 | 绑定按键、调节长按时长 |
| 自定义循环插件 IPC 路径 | 手动填入状态/暂停/恢复的 IPC 全路径 |
| 调试模式 | 启用调试输出、调节输出间隔 |
| 自定义陌语命令 | 自由增删改命令及其绑定动作 |

## 🔧 常见问题

**Q: 插件加载后不显示？**
A: 检查 `AutoFollow.json` 中的 `DalamudApiLevel` 是否匹配你的卫月版本。国服 API 15 用 `"DalamudApiLevel": 15`。

**Q: 跟随帧率骤降？**
A: 检查是否打开了调试模式，调试输出间隔默认 2 秒，不会影响帧率。如果仍卡顿，检查是否有其他插件冲突。

**Q: 循环插件不自动暂停？**
A: 先确认循环插件已安装且正在运行。如果循环插件不是 AutoDuty，需要在 **自定义循环插件 IPC 路径** 中填入正确的 IPC 名称。

**Q: 怎么切换跟随目标？**
A: 再次使用 `/ft <新玩家名>` 即可切换，或 `/ft` 不加参数清除目标。
